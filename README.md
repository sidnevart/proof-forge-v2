# Proof-Forge v2

Agent-native learning workspace. Ты учишь тему, а AI-агент (Claude) строит капсулу знаний, задаёт вопросы и отслеживает слабые места. Бэкенд хранит данные, агент думает сам.

---

## Установка

Нужен [`uv`](https://docs.astral.sh/uv/) (`curl -LsSf https://astral.sh/uv/install.sh | sh`).

### 1. Установи плагин

```bash
uv tool install "git+https://github.com/sidnevart/proof-forge-v2.git#subdirectory=apps/mcp-server"
```

Проверь путь к бинарнику: `which proof-forge-mcp`

### 2. Добавь в конфиг Claude Desktop

Файл: `~/Library/Application Support/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "proof-forge": {
      "command": "proof-forge-mcp",
      "env": {
        "BACKEND_URL": "https://api.proof-forge.ru"
      }
    }
  }
}
```

> Если Claude Desktop не видит команду — подставь полный путь из `which proof-forge-mcp`.

**3.** Перезапусти Claude Desktop.

Всё. Больше никакой настройки — при первом использовании Claude спросит email и создаст профиль.

---

### Claude Code (CLI)

```bash
claude mcp add proof-forge -s user \
  -e BACKEND_URL=https://api.proof-forge.ru \
  -- proof-forge-mcp
```

## Первое использование

Один раз скажи свой email — и профиль будет доступен с любого устройства:

```
identify me as your@email.com
```

Claude вызовет `identify(email)` → создаст профиль на сервере → сохранит в `~/.proof-forge/identity`. На следующих устройствах достаточно повторить эту команду с тем же email.

Если забудешь — Claude сам спросит email при первой попытке что-то сделать.

## Как пользоваться

### Учиться

```
помоги изучить Kotlin Coroutines
```
→ Claude объясняет тему как ментор (study-mentor-v2), задаёт вопросы, пишет конспект

```
закончил
```
→ Тема закрыта, форжится капсула знаний, создаются карточки для повторения

### Повторять

```
карточки
```
→ Claude показывает карточки которые пора повторить сегодня (SM-2 алгоритм)

```
review
```
→ Интерактивный quiz: Claude достаёт твои капсулы, задаёт вопросы, оценивает ответы

### Отслеживать прогресс

```
прогресс
```
→ Claude показывает уровень освоения каждого концепта (🟥 не начат → 🟨 узнаю → 🟩 применяю → 🟦 объясняю), что отработано и **что осталось до эксперта**

### Фиксировать

```
конспект
```
→ Структурированный конспект из текущего материала → сохраняется в Proof-Forge

```
forge capsule
```
→ Полная капсула знаний: Replay, Concept Map, слабые места, вопросы для ревью

## Прогресс к эксперту

Система измеряет освоение **каждого концепта** темы, а не просто «прошёл / не прошёл». Уровни:

| Бейдж | Уровень | Как достигается |
|---|---|---|
| 🟥 | не начат | — |
| 🟨 | узнаю | теория объяснена |
| 🟩 | применяю | ≥2 практики, качество ≥0.6, сложность ≥2 |
| 🟦 | объясняю | ≥3 практики, качество ≥0.8, сложность ≥3, пройден struggle-check |

**«Эксперт по теме» = все ключевые концепты на 🟦.** Это измеримый порог, а не лозунг. Напиши `прогресс` чтобы увидеть где ты и что доработать.

Сложность заданий калибруется под тебя: если «сложное» задание далось легко и ты прошёл struggle-check (follow-up на edge-case / trade-off) — Claude поднимает планку. Цель — продуктивная трудность, а не лёгкие победы.

## Карточки — как работают

Карточки создаются автоматически при сохранении капсулы. Каждый вопрос из раздела "Вопросы для ревью" становится одной карточкой.

**Когда появляются:** первое повторение — на следующий день после создания капсулы. Дальше интервал управляется алгоритмом SM-2:

| Оценка | Что происходит |
|---|---|
| 1 — Не помнил | Снова завтра |
| 2 — С трудом | Через несколько дней |
| 3 — Нормально | Интервал растёт: 1 → 6 → ~15 → ~38 дней |
| 4 — Легко | Интервал растёт быстрее |

**Как посмотреть:** карточки появляются только когда ты спрашиваешь — напиши `карточки` и Claude покажет всё что накопилось к сегодняшнему дню. Push-уведомлений нет (планируется в Telegram-боте).

**Типичный ритм:** после изучения темы → на следующий день пишешь "карточки" → проходишь 5–10 минут ревью → интервалы разъезжаются и карточки возвращаются реже.

## Капсула — где и как посмотреть

Капсула хранится на сервере как markdown. Содержит: краткое описание темы, концепт-карту, слабые места, код-карту, вопросы с ответами и следующие шаги.

**В чате с Claude:**
```
покажи капсулу по Kotlin Coroutines
```
Claude вызовет `get_agent_context` → найдёт нужную капсулу → выведет полный текст прямо в чат.

**Через браузер (Swagger UI):**

`https://api.proof-forge.ru/docs` → раздел `capsules` → `GET /api/capsules/{capsule_id}`

Капсула возвращается в двух форматах:
- `content_md` — исходный markdown
- `content_html` — готовый HTML для отображения

**Через curl:**
```bash
curl https://api.proof-forge.ru/api/capsules/{capsule_id}
```

`capsule_id` Claude всегда сообщает после сохранения: _"Капсула сохранена (capsule_id: ...)"_.

## Все команды

| Что написать | Что происходит |
|---|---|
| `помоги изучить <тема>` | Ментор объясняет тему, пишет конспект |
| `закончил` / `wrap up` | Закрывает тему, форжит капсулу, создаёт карточки |
| `forge capsule` | Полная капсула + карточки из сессии |
| `конспект` / `напиши summary` | Структурированный конспект |
| `карточки` / `anki` / `повторить` | Ежедневное повторение по SM-2 |
| `review` / `проверь знания` | Quiz с разбором по капсулам |
| `check my answer: <ответ>` | Оценить конкретный ответ |
| `analyze my code` | Разобрать код, сохранить артефакт |
| `export context` | Весь прогресс — профиль, капсулы, слабые места |
| `identify me as email@example.com` | Привязать email к аккаунту |

### Learning loop

```
"помоги изучить Kotlin Coroutines"
  └─ объясняет → конспект → store_capsule

"закончил"
  └─ complete_topic → капсула → create_cards_from_capsule

"карточки"  ← на следующий день
  └─ get_due_cards → quiz → log_card_attempt (SM-2 интервал)

"review"
  └─ get_agent_context → get_capsule → quiz → store_review_answer
```

## Self-hosted

**Требования:** Docker, Python 3.12+, `uv`

```bash
git clone https://github.com/sidnevart/proof-forge-v2
cd proof-forge-v2
cp .env.prod.example .env.prod
# задай POSTGRES_PASSWORD в .env.prod
docker compose -f docker-compose.v2.yml up --build
```

Backend: `http://localhost:8001` — измени `BACKEND_URL` в конфиге плагина.

API docs: `http://localhost:8001/docs`

```bash
./scripts/test-local.sh           # unit + evals
./scripts/test-local.sh --no-evals  # только unit
```

## Архитектура

```
Ты (в чате с Claude)
  └── Claude Desktop / Claude Code
        ├── packages/skills/*.md — скиллы с триггерными фразами
        └── MCP инструменты (13 tools)
              └── proof-forge-mcp  ← установлен локально
                    └── HTTPS → api.proof-forge.ru
                                  └── PostgreSQL
```

Агент делает всё мышление. Бэкенд — чистое хранилище.

| Путь | Что это |
|---|---|
| `apps/backend/` | FastAPI + PostgreSQL, SM-2 spaced repetition |
| `apps/mcp-server/` | MCP сервер, 13 инструментов, авто-identity |
| `packages/skills/` | 11 скиллов с триггерными фразами |
| `tests/` | pytest + evals через Ollama |
