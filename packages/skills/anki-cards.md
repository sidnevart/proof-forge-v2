# Anki Cards

> **Педагогика — [_pedagogy.md](_pedagogy.md):** при повторении не показывай ответ сразу — если ученик колеблется, дай hint (направление), потом ответ. После ответа на карточку вызывай `record_mastery`.

Два режима. Определи по контексту какой нужен:

- **Режим А — Повторение**: пользователь хочет повторить уже сохранённые карточки ("карточки", "повторить")
- **Режим Б — Сбор**: пользователь хочет создать карточки из нового материала ("собери карточки из ...", "сделай anki из ...")

---

## Режим А — Ежедневное повторение

### Шаг 1: Загрузи карточки к повторению

```
get_due_cards(limit=10)
```

Если список пустой:
```
На сегодня карточек нет. Следующее повторение: [дата ближайшей карточки из предыдущих попыток].
Хочешь начать новую тему? Попробуй: "помоги изучить [тема]"
```

Если карточки есть — сообщи: _"Сегодня к повторению: N карточек. Поехали!"_

### Шаг 2: Веди сессию карточка за карточкой

Для каждой карточки:

1. Покажи вопрос:
```
🃏 Карточка [номер]/[всего]  |  Тема: [topic_name]  |  Сложность: [difficulty]/3

[question]

(Напиши ответ своими словами)
```

2. Пользователь отвечает.

3. Оцени ответ сам (0.0–1.0), не спрашивай пользователя оценить себя — делай это ты.

4. Покажи правильный ответ и обратную связь:
```
✅ Правильный ответ: [correct_answer]

[Твоя оценка ответа пользователя — что правильно, что упущено]
```

5. Спроси насколько легко вспомнилось (это для SM-2 интервала):
```
Насколько легко вспомнил?
1 — Не помнил совсем (повторить завтра)
2 — С трудом (интервал слегка увеличится)
3 — Вспомнил нормально (стандартный интервал)
4 — Легко (интервал увеличится быстрее)
```

6. Вызови:
```
log_card_attempt(card_id, rating=<1-4>, user_answer=<ответ пользователя>)
record_mastery(topic_id, concept=<концепт карточки>, kind="practice", difficulty=<difficulty карточки>, quality_score=<твоя оценка 0..1>)
```

7. После успешной записи — переходи к следующей карточке без лишних комментариев.

### Шаг 3: Итог сессии

```
🎯 Повторение завершено!

Карточек сегодня: N
Хорошо (3-4): X  |  Трудно (1-2): Y

[Если были трудные] Повторить завтра: [список тем где rating 1-2]

Следующее повторение: [ближайшая дата из log_card_attempt ответов]
```

## Режим Б — Сбор карточек из материала

Если пользователь хочет создать карточки из заметок / кода / темы без полного учебного цикла.

### Шаг 1: Проверь есть ли уже капсула

Если у пользователя есть `capsule_id`:
```
create_cards_from_capsule(capsule_id)
```
→ Карточки созданы из вопросов капсулы. Готово.

Если нет капсулы — работай с предоставленным материалом:

### Шаг 2: Выдели атомарные факты

Из предоставленного материала (заметки / код / тема):
- Каждый атомарный факт = одна карточка
- Карточка: вопрос (одна сторона) ↔ ответ (другая сторона)
- Идеальная карточка: вопрос на одно понятие, ответ 1-3 предложения

**Атомарность — одна карточка = одна мысль.** Примеры:
- ✅ «Чем отличается val от var в Kotlin?» (один чёткий контраст)
- ❌ «Расскажи про переменные в Kotlin» (слишком широко)
- ✅ «Что вернёт `listOf(1,2,3).firstOrNull { it > 5 }`?» (один факт, проверяемо)
- ❌ «Как работают коллекции в Kotlin?» (это тема, не карточка)
- ✅ «Зачем нужен `@Volatile` для поля, читаемого из нескольких потоков?» (один механизм)

Формат хорошей карточки:
```
Q: Чем отличается val от var в Kotlin?
A: val — неизменяемая ссылка (аналог final в Java), var — изменяемая.
   Оба могут быть nullable через ?.
```

### Шаг 3: Создай минимальную капсулу и карточки

Если нет существующей капсулы:
1. Собери вопросы из материала
2. Вызови `store_capsule(topic_id, content_md, summary, review_questions)` — минимальная капсула
3. Вызови `create_cards_from_capsule(capsule_id)`
4. Сообщи: _"Создано N карточек. Первое повторение завтра."_

---

## Правила

- В Режиме А не показывай правильный ответ ДО того как пользователь ответит
- Оценивай ответ сам — не проси пользователя нажимать "правильно/неправильно"
- Рейтинг 1-4 определяет интервал SM-2: 1=завтра, 2=скоро, 3=стандартно, 4=долго
- Максимум 10 карточек за сессию, потом предложи паузу
- Если пользователь говорит "хватит" / "стоп" в любой момент — завершай сессию и показывай итог
